## Overview

A personal portfolio and blog built with Jekyll and the Minimal Mistakes
theme, hosted on GitHub Pages. The site documents my projects, technical
writing, and professional work.

## Problem Statement

I needed a place to document and share my work — projects, published
research, and technical notes — without the overhead of managing a server
or paying for hosting.

## Approach

Used Jekyll with the Minimal Mistakes dark theme. GitHub Pages builds and
deploys the site automatically on every push to `main`. Content is written
in Markdown. A structured `_brainstorm/` directory (gitignored) manages
ideas and session-to-session context for Claude Code collaboration.

## Stack

| Technology | Why |
|-----------|-----|
| Jekyll | GitHub Pages builds it automatically; no CI needed |
| Minimal Mistakes | Dark theme, portfolio collection support, best docs |
| GitHub Pages | Free hosting, custom domain support, automatic deploys |
| Markdown | Low friction; same format used for brainstorming |

## Outcome

Site is live and iterating. Future phases add assets, SCSS customization,
resume, and published works.
